# Coin Change with Dynamic Programming

**Language:** JavaScript
**Difficulty:** Medium

## Problem
Given coin denominations and a target amount, return the minimum number of coins needed to form that amount.

## Solution
```javascript
function coinChange(coins, amount) {
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0;

  for (let value = 1; value <= amount; value += 1) {
    for (const coin of coins) {
      if (coin <= value) {
        dp[value] = Math.min(dp[value], dp[value - coin] + 1);
      }
    }
  }

  return Number.isFinite(dp[amount]) ? dp[amount] : -1;
}
```

## Explanation
Dynamic programming works well here because every amount depends on smaller amounts. We build the answer bottom-up, using `dp[value]` to store the fewest coins needed for each value.

For each amount, we try every coin. If using that coin leads to a better answer than what we already have, we update the table. By the end, the last entry contains the minimum coin count or remains unreachable.

## Key Points
- Define `dp[value]` as the best answer for that subproblem.
- Build from smaller amounts to larger ones.
- Bottom-up DP avoids repeated recalculation.
