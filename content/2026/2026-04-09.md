# Coin Change Problem with Memoization

**Language:** JavaScript  
**Difficulty:** Hard

## Problem

Given a target sum and a set of coin denominations, calculate the number of ways to make the target sum using the given coins. You can use each coin any number of times.

## Solution

```javascript
function coinChange(coins, amount) {
  const memo = new Map();
  function dp(n) {
    if (n in memo) return memo.get(n);
    if (n === 0) return 1;
    if (n < 0) return 0;
    let res = 0;
    for (const coin of coins) {
      res += dp(n - coin);
    }
    memo.set(n, res);
    return res;
  }
  return dp(amount);
}

// Test the function
console.log(coinChange([1, 2, 5], 5));
console.log(coinChange([2], 3));
console.log(coinChange([1], 0));
console.log(coinChange([], 4));
```

## Explanation

The Coin Change problem is a classic dynamic programming problem. We can solve it using memoization to avoid redundant calculations. The idea is to store the results of subproblems in a map (memo) and reuse them instead of recalculating them. The time complexity of this solution is O(n * m), where n is the target sum and m is the maximum coin denomination. The space complexity is also O(n * m) for storing the memo.

- Dynamic programming problems can be solved using recursion, but memoization is often more efficient.
